# 目标规划

## 当前基线：0.3.0-dev

- DNS、Mihomo、OpenVPN、system/service、firewall 模块已拆分；
- `doctor router` 通过一次 SSH聚合模块；
- 单模块命令只校验自身必填配置；
- 退出状态区分诊断故障、配置错误与 SSH传输失败（0/1/2/3）；
- GitHub Actions CI 与 main branch protection 已启用；
- fixture/stub 覆盖 `[OK]`、`[WARN]`、`[!]`、fw3/fw4 与缺命令场景；
- OpenWrt/GL.iNet 真实设备只读基线在 0.2.0 验证；0.3.0 新模块的真机只读验证待维护者执行；
- 项目采用 Apache-2.0。

## 0.3.0-dev：稳定模块契约

在增加结构化输出前，先完成以下任务。

> 实现状态：下列 P0/P1 项已全部合并进 main；真机只读验证与发布检查由维护者执行。评审遗留的低优先级改进见 0.4.0-dev。

### P0：按命令校验配置 — 已实现

目标：单模块命令只要求自身所需配置，避免 `doctor dns` 因缺少 VPN参数而失败。

验收：

- [x] 为每个命令定义明确的必填配置集合；
- [x] 未知配置项和注入字符继续被拒绝；
- [x] `doctor router` 仍验证全部配置；
- [x] 增加最小配置、缺项和恶意值测试。

### P0：防火墙模块 — 已实现

目标：补齐 DNS → Fake-IP → 路由 → Clash规则 → VPN → 防火墙 → 最终服务证据链。

验收：

- [x] 新增只读 firewall 模块并纳入 `doctor router`；
- [x] 兼容常见 fw3/iptables 与 fw4/nftables 能力探测；
- [x] 不硬编码私人接口名或区域名；
- [x] 工具缺失时输出 `[WARN]`，明确规则缺失时输出 `[!]`；
- [x] fixture覆盖 fw3、fw4、缺命令和缺规则。

### P1：执行错误分类 — 已实现

目标：区分本地配置错误、SSH连接错误、远程能力缺失和明确诊断失败。

验收：

- [x] 文档化退出状态；
- [x] SSH失败不会伪装成模块 `[!]`；
- [x] 超时、缺命令和不可读运行配置有稳定判断；
- [x] 聚合模式尽可能完成所有安全检查后再返回。

### P1：持续集成 — 已实现

目标：所有公开变更自动执行 Shell语法、ShellCheck、fixture和敏感模式检查。

验收：

- [x] GitHub Actions 使用最小权限；
- [x] CI只使用仓库脱敏数据，不配置真实 SSH凭据；
- [x] 本地 `make check` 与 CI入口一致；
- [x] main分支合并前要求 CI通过（branch protection 已启用：必需检查 make check，含管理员）。

## 0.4.0-dev：结构化结果

### 评审遗留（0.3.0 评审 warn，优先处理）

- [ ] 扩充 `scripts/check-sensitive.sh` 覆盖面：大小写不敏感匹配、YAML/JSON 冒号分隔的 secret 赋值、`ss://`/`hysteria2://`/`tuic://` 等代理协议、`ghp_`/`AIza`/URL 内嵌凭据等令牌形式；

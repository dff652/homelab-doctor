# 最终 Review 清单

开发完成后，以开始本轮开发时记录的 `main` 基线为起点检查新增提交。评审以实际代码、测试和只读集成证据为准，不只依据开发人员的完成说明。

## 1. 交付范围

- [ ] 按命令进行最小配置校验；
- [ ] 新增只读 firewall 模块；
- [ ] 兼容 fw3/iptables 与 fw4/nftables；
- [ ] 区分配置错误、SSH错误、能力缺失和明确诊断失败；
- [ ] 建立 GitHub Actions CI；
- [ ] README、架构和开发文档同步；
- [ ] 未提前实现 JSON/JUnit、Web UI或自动修复。

## 2. 安全边界

- [ ] 不执行重启、清缓存或配置写入；
- [ ] 不包含 `uci set`、`uci commit`、防火墙修改等隐式修复；
- [ ] 不 `source` 用户配置；
- [ ] 所有进入远程命令的参数均经过白名单验证；
- [ ] SSH参数不存在命令注入路径；
- [ ] 每个诊断命令只建立一次 SSH连接；
- [ ] 不在路由器安装文件或常驻 Agent；
- [ ] 当前树及完整 Git历史没有真实地址、域名、token、订阅、私钥或事故日志。

## 3. 配置校验

- [ ] `doctor dns` 只要求 DNS相关配置；
- [ ] `doctor mihomo` 只要求 Mihomo相关配置；
- [ ] `doctor openvpn` 只要求 VPN相关配置；
- [ ] `doctor router` 要求所有聚合模块配置；
- [ ] 未知配置项被拒绝；
- [ ] 缺少必填项时输出明确错误；
- [ ] 分号、引号、反引号、换行等注入字符被拒绝；
- [ ] 示例配置保持完全脱敏。

## 4. Firewall 模块

- [ ] 存在 fw3/iptables 健康 fixture；
- [ ] 存在 fw4/nftables 健康 fixture；
- [ ] 存在转发规则缺失 fixture；
- [ ] 存在相关命令缺失 fixture；
- [ ] 不硬编码私人接口名、zone名或设备地址；
- [ ] 能力缺失输出 `[WARN]`；
- [ ] 明确规则缺失输出 `[!]`；
- [ ] 已纳入 `doctor router` 聚合顺序和摘要。

## 5. 输出与退出状态

- [ ] `[OK]` 只用于明确健康证据；
- [ ] `[WARN]` 用于非阻断风险或无法检查；
- [ ] `[!]` 用于明确故障；
- [ ] 仅有警告时返回 0；
- [ ] 存在故障时返回非零；
- [ ] SSH失败不会伪装成远程模块故障；
- [ ] `SUMMARY ok=N warn=N fail=N` 计数准确；
- [ ] 聚合模式在安全范围内尽可能完成全部检查。

## 6. 兼容性

- [ ] 远程脚本通过 `sh -n`；
- [ ] ShellCheck通过；
- [ ] 不使用 Bash专属语法；
- [ ] 兼容 OpenWrt BusyBox工具输出；
- [ ] 缺少 `dig`、`curl`、`netstat`、`iptables`或`nft`时安全降级；
- [ ] 不增加不必要的路由器运行时依赖。

## 7. 自动测试与 CI

- [ ] `make check` 通过；
- [ ] `git diff --check` 通过；
- [ ] 所有模块覆盖健康、警告和故障状态；
- [ ] 存在单模块最小配置测试；
- [ ] 存在配置缺项和注入测试；
- [ ] 存在 SSH连接失败与超时测试；
- [ ] `doctor router` 保持单次 SSH调用断言；
- [ ] 存在 fw3、fw4及缺命令测试；
- [ ] CI使用最小权限；
- [ ] CI不包含真实 SSH凭据或私有配置。

## 8. 真实设备验证

- [ ] 使用 gitignored 的临时配置；
- [ ] 对 OpenWrt/GL.iNet设备执行只读测试；
- [ ] 分别测试新增模块与 `doctor router`；
- [ ] 评审记录只保留脱敏计数和退出状态；
- [ ] 测试完成后删除临时配置；
- [ ] 集成测试后重新扫描敏感信息。

## 9. 文档与仓库状态

- [ ] README命令与实际 CLI一致；
- [ ] 架构文档包含新模块与执行顺序；
- [ ] Roadmap完成状态已更新；
- [ ] 新增行为有明确限制说明；
- [ ] 提交聚焦且可独立审查；
- [ ] PR或提交没有无关格式化；
- [ ] 工作区干净；
- [ ] 本地与远程 `main` 同步。

## 最终结论

- **通过**：所有关键项满足，可以进入下一阶段；
- **有条件通过**：仅剩文档或低风险测试债，不影响安全边界；
- **阻断**：存在写操作、注入风险、敏感信息、退出状态错误、BusyBox不兼容或测试失败。

阻断项必须修复并重新执行相关检查，不能通过备注豁免只读与安全约束。
